# 旅游路线地图功能实现计划

## 需求概述

当用户完成旅游规划后，询问是否需要地图路线导航：
- 用户输入起始地址
- 系统将起始地址 + 规划路线中的所有景点按顺序连接
- 在前端页面展示路线图

---

## 整体流程

```
用户: "我想去广州旅游3天"
    ↓
Agent 生成旅游规划
    ↓
用户确认/不满意 → 调整 → 直到满意
    ↓
询问: "需要显示地图路线吗？"
    ↓ 用户选择 "需要"
    ↓
用户输入起始地址
    ↓
Agent 调用高德地图API获取各景点坐标
    ↓
生成路线规划（驾车/步行/公交）
    ↓
前端展示地图路线
```

---

## 需要实现的功能模块

### 1. 后端 - 新增 API 接口

**文件**: `travel_ai_python/app/web/api.py`

新增接口：

```python
@router.post("/route/map")
async def get_map_route(request: MapRouteRequest):
    """
    获取地图路线
    请求:
      - start_address: 起始地址
      - destinations: [景点1, 景点2, ...]
      - travel_mode: "driving" | "walking" | "transit"
    返回:
      - 路线规划详情
      - 各站点坐标
      - 路线距离和时间
    """
```

---

### 2. 高德地图 API 集成

**文件**: `travel_ai_python/app/utils/gaode_map.py`（新建）

需要实现：

| 功能 | 高德API | 说明 |
|------|---------|------|
| 地址转坐标 | `https://restapi.amap.com/v3/geocode/geo` | 输入地址→经纬度 |
| 驾车路线规划 | `https://restapi.amap.com/v5/direction/driving` | 多站点最优路线 |
| 步行路线规划 | `https://restapi.amap.com/v5/direction/walking` | 步行导航 |
| 公交路线规划 | `https://restapi.amap.com/v5/direction/transit` | 公共交通 |

**关键逻辑**：
1. 将起始地址和所有景点地址转为坐标
2. 调用高德路径规划 API，传入途经点顺序
3. 返回路线详情（距离、时间、拐点坐标）

---

### 3. Agent 对话逻辑修改

**文件**: `travel_ai_python/app/agents/planner.py` 或 `final_optimizer.py`

在旅游规划确认满意后，新增流程：

```python
# 规划完成后
if user_confirmed:
    # 询问是否需要地图
    ask_map_guidance = "您已确认旅游规划。是否需要显示地图路线导航？"
    return ask_map_guidance
```

用户选择需要后，调用地图路线接口。

### 4. 前端地图组件

**文件**: 前端项目（需确认前端技术栈）

推荐方案：

| 方案 | 库 | 优点 |
|------|-----|------|
| 高德 Web端 | AMap JS API 2.0 | 高德官方，数据准确 |
| Mapbox | mapbox-gl-js | 样式美观 |
| Leaflet | leaflet | 轻量级 |

**前端需要实现**：
1. 地图组件（显示起点、终点、途经点）
2. 路线绑点（连接各景点形成路线）
3. 信息弹窗（显示每个点的名称和顺序）

## 数据结构设计

### MapRouteRequest

```python
class MapRouteRequest(BaseModel):
    start_address: str  # 起始地址，如 "广州市天河区正佳广场"
    destinations: list[str]  # 景点列表，如 ["广州塔", "陈家祠", "白云山"]
    travel_mode: str = "driving"  # driving/walking/transit
```

### MapRouteResponse

```python
class MapRouteResponse(BaseModel):
    success: bool
    route_info: {
        "total_distance": 12500,  # 米
        "total_duration": 1800,    # 秒
        "steps": [
            {
                "from": {"name": "起点", "lat": 23.123, "lng": 113.321},
                "to": {"name": "广州塔", "lat": 23.456, "lng": 113.789},
                "distance": 3500,
                "duration": 600
            },
            ...
        ]
    }
    waypoints: [  # 用于前端绑点
        {"name": "起点", "lat": 23.123, "lng": 113.321},
        {"name": "广州塔", "lat": 23.456, "lng": 113.789},
        ...
    ]
```

## 高德地图 API 申请步骤

1. 访问 [高德开放平台](https://lbs.amap.com/)
2. 注册开发者账号（个人开发者免费）
3. 创建应用 → 添加 Key（Web服务 JS API）
4. 设置 IP 白名单（你的服务器 IP）
5. 复制 Key 到 `.env` 文件

---

## 注意事项

1. **高德 API 免费额度**: 5000次/日，超出按量计费
2. **路线规划限制**: 驾车路线最多16个途经点，需合理切分
3. **坐标转换**: 部分地址可能无法识别，需要容错处理
4. **缓存策略**: 对相同地址的坐标转换结果进行缓存，减少API调用
