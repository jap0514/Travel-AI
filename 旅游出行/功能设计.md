## 一、用户端（微信小程序）功能

| 功能点           | 说明                                                         |
| ---------------- | ------------------------------------------------------------ |
| 微信授权登录     | 调用 `wx.login` 获取 code，后端换取 openId 并生成 JWT Token  |
| 文字输入旅游需求 | 用户手动输入文字描述旅游想法，提交生成攻略                   |
| 语音输入旅游需求 | 通过微信录音接口获取音频，调用语音识别 API 转为文本后提交    |
| 实时查看任务进度 | 通过 WebSocket（STOMP）订阅任务状态，接收“排队中/处理中/完成”等推送 |
| 获取完整行程结果 | 任务完成后展示结构化的旅游规划：每日行程、景点、美食、文化描述、交通方式、天气 |
| 历史行程列表     | 查看用户以往生成的行程记录（分页）                           |
| 删除行程记录     | 删除某条历史行程及关联的细节数据                             |
| 个人信息管理     | 修改昵称、头像、预算偏好、行程节奏偏好（休闲/紧凑等）        |
| 对话历史查看     | 查看与 AI 模型的聊天记录（支持按会话 sessionId 查询）        |
| 补充对话/追问    | 针对已生成的行程进行追问（如“增加一天”），支持上下文关联     |

## 二、后台管理端功能

| 功能点                   | 说明                                                         |
| ------------------------ | ------------------------------------------------------------ |
| 管理员登录               | 独立的管理端登录接口，返回 JWT + 角色权限                    |
| 角色管理（RBAC）         | 增删改查角色，分配菜单权限                                   |
| 菜单管理                 | 管理后台菜单树结构，支持拖拽排序、隐藏/显示                  |
| 用户管理                 | 查看所有微信用户列表，禁用/启用账户，重置密码（仅限管理员）  |
| Token 消耗监控           | 统计所有用户或单个用户的 Token 使用量（按天/小时），支持图表展示 |
| 调用正确率监控           | 统计任务成功率（COMPLETED 数 / 总任务数），失败原因分布      |
| 系统健康检查             | 一键检测 Java、Python、RabbitMQ、MySQL、Redis 状态并返回结果 |
| 全链路日志检索           | 按 traceId、userId、taskId、时间范围、日志级别检索日志       |
| 业务统计看板             | 展示热门搜索关键词（景点/城市）、任务请求量趋势、平均响应耗时、Python 处理耗时 |
| 知识库管理               | 上传 PDF/Markdown 文档，触发向量化；查看已向量化文档列表；删除文档及对应向量 |
| 向量检索测试             | 输入文本，测试向量数据库检索结果（用于调试 RAG 效果）        |
| 动态配置管理             | 在线修改大模型 Agent 的 System Prompt、温度参数、RAG 召回数量等（无需重启） |
| 任务补偿接口             | 手动重试失败任务（POST /api/task/retry/{taskId}）或取消正在执行的任务 |
| Redis 数据管理（管理员） | 查看/删除 Redis 中的缓存键（如任务超时 key、会话 key）       |
| 向量数据库管理           | 删除指定 ID 的向量数据，清空整个知识库集合（需二次确认）     |

## 三、系统核心功能（后端内部）

| 功能点                      | 说明                                                         |
| --------------------------- | ------------------------------------------------------------ |
| 用户认证与权限校验          | Spring Security + JWT，基于角色的 RBAC 拦截器                |
| 任务状态机                  | 管理任务生命周期：INIT → QUEUED → PROCESSING → COMPLETED / FAILED |
| 任务持久化                  | MySQL `task` 主表存储任务状态、时间、用户关联                |
| 结果结构化存储              | 多表存储：`trip_plan`、`daily_schedule`、`attractions`、`foods`、`transport`、`weather` |
| RabbitMQ 消息发送与确认     | Java 将任务 JSON 发送至交换机，开启 Publisher Confirm        |
| RabbitMQ 消息消费（Java侧） | 监听结果队列，接收 Python 回传的进度和最终结果               |
| WebSocket 实时推送          | 基于 STOMP 协议，绑定 userId 和 taskId，异步推送进度和结果   |
| Redis 超时控制              | 任务开始时存入 Redis（key = `task:timeout:{taskId}`，TTL 60s），超时则标记失败 |
| Python 大模型编排           | 意图识别、推理决策、工具调用链（ReAct 或 Function Calling）  |
| 技能执行                    | 调用外部 API：天气查询（高德/和风）、景点搜索（百度/高德 POI）、美食推荐、路线规划（高德导航） |
| RAG 检索增强                | 基于向量数据库（如 Milvus/Chroma）检索相关攻略文档，注入 Prompt |
| 格式化输出                  | Python 将结果整理为约定 JSON 结构，发送至 RabbitMQ           |
| 全链路日志追踪              | Java 生成 traceId 放入 MDC，通过 RabbitMQ 消息头传递给 Python，Python 日志同样打印 |
| 异常处理 - 通信             | RabbitMQ 发送失败重试 3 次，失败后状态变为 FAILED            |
| 异常处理 - 业务             | AI 调用超时触发 Redis 超时标记；MQ 重复消息通过 taskId 幂等检查 |
| 异常处理 - Python 侧        | 模型返回格式错误时发送错误消息，Java 捕获后状态 FAILED       |
| 死信队列                    | 重试 3 次仍失败的消息进入死信队列，人工介入                  |
| 异步补偿接口                | 提供 REST API 手动重试、取消任务（修改状态 + 发送取消信号给 Python） |

## 四、数据管理功能

| 功能点       | 涉及存储              | 说明                                                         |
| ------------ | --------------------- | ------------------------------------------------------------ |
| 用户表       | MySQL                 | `user`：id, openId, nickname, avatar, preferences(JSON)      |
| 任务表       | MySQL                 | `task`：taskId, userId, query, status, traceId, create_time, complete_time |
| 行程主表     | MySQL                 | `trip_plan`：planId, taskId, title, totalDays, budget, pacing |
| 每日计划表   | MySQL                 | `daily_schedule`：dayIndex, morning_activity, afternoon_activity, evening_activity, hotel |
| 景点表       | MySQL                 | `attractions`：name, address, lat, lng, visit_duration, ticket_price |
| 美食表       | MySQL                 | `foods`：name, cuisine, price_per_person, recommended_dish   |
| 日志表       | MySQL / Elasticsearch | `log`：traceId, userId, level, message, timestamp            |
| 知识库文档表 | MySQL                 | `knowledge_doc`：docId, fileName, uploadTime, vectorCount    |
| 向量数据     | 向量数据库            | 存储文档切片 embedding，用于 RAG 检索                        |
| 任务超时键   | Redis                 | `task:timeout:{taskId}`，TTL 60 秒                           |
| 用户会话     | Redis                 | 存储 JWT 黑名单、WebSocket 会话映射（userId -> sessionId）   |
| 热门搜索统计 | Redis                 | 使用 ZSET 记录用户搜索词频（用于看板）                       |
