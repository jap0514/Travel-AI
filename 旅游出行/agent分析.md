# 加入 Mem0 后｜Agent V1~V4 分阶段落地规划（清晰分点版）
## V1 阶段：核心底座 **Memory + Mem0 + MCP**
1. 框架选型
- LangChain 生态最成熟；
- 做复杂流程、循环分支直接上 **LangGraph**，后续工作流天然适配。

2. 核心组件选型
- LLM：选用 GPT-4o、DeepSeek；
- 短期会话记忆：**Redis** 存当前会话上下文；
- 长期用户记忆：接入 **Mem0** 做用户偏好、历史习惯自动记忆与召回；
- 工具标准层：**MCP 官方 Python SDK** 统一标准化工具调用；
- 业务工具：普通 Python 函数实现（查天气、查酒店、算距离、查景点）。

3. V1 完整业务流程
- 用户请求 → Java → RocketMQ → Python 服务；
- Python 先根据 **sessionId** 从 Redis 加载本轮会话短期上下文；
- 同时通过 **Mem0** 根据 **userId** 拉取用户长期偏好记忆（出行习惯、酒店喜好、预算倾向）；
- Agent 结合「会话上下文 + Mem0 用户记忆」做决策；
- 需要外部能力时，通过 MCP 调用对应工具 API；
- 工具返回结果交给 LLM 整合生成回答；
- 自动把本轮对话、用户隐性偏好**写入 Mem0 做记忆沉淀**，同时会话记录落 Redis；
- 最终结果经 RocketMQ 返回 Java，入库并推给前端。

---

## V2 阶段：能力编排 **Workflow + Skills**
1. 核心载体：依旧用 **LangGraph**，天然支持多节点编排、状态流转。
2. 能力拆分为可复用 **Skill**
- 独立原子技能：目的地提取、景点查询、距离计算、酒店查询、预算解析等；
- 每个 Skill 可单独开发、单独调试、复用在不同流程里。
3. Workflow 工作流编排
- 用 LangGraph 把多个 Skill 按业务串成完整流程；
- 示例：参数提取 → 并发查天气/酒店 → 生成初版行程 → 距离合规校验；
- LangGraph 统一管理全局状态，只需定义每个节点业务逻辑即可。
4. Mem0 作用
- Workflow 每走完一轮，自动提炼用户偏好存入 Mem0；
- 下次同用户新请求，直接复用记忆，不用重复问相同偏好问题。

---

## V3 阶段：智能自愈 **Loop + Evaluation**
1. 核心价值：让 Agent 具备**自我评估、循环修正**能力。
2. Evaluation 评估方案
- 两种方式：LLM 自评打分 / 自定义评分函数；
- 评估维度：预算合理性、距离适配度、用户偏好满足度、行程流畅度。
3. Agent Loop 循环机制
- 在 LangGraph 中配置循环判断节点；
- 生成行程方案后先做 Evaluation 打分；
- 低于阈值（如80分）则进入循环：告知 LLM 短板 → 重新调整方案 → 再次打分；
- 直到分数达标 或 达到最大重试次数，退出循环。
4. Mem0 联动
- 评估优秀的方案，提炼用户喜好固化到 Mem0；
- 低分不合理方案，不沉淀、避免污染长期记忆。

---

## V4 阶段：个性化增强 **向量数据库 + 长期记忆融合**
1. 接入时机：业务跑通、有一定用户历史数据后再引入，不用过早搭建。
2. 向量库选型
- 轻量起步：Chroma；
- 生产商用：Qdrant / Milvus。
3. 架构分工（Mem0 + 向量库 各司其职）
- **Mem0**：负责**用户偏好、人设、习惯**的结构化长期记忆，自动增删、自动遗忘；
- **向量数据库**：负责**历史行程、优质案例、知识库文档**的非结构化检索。
4. 业务流程
- 将用户历史行程、优秀方案、攻略文档向量化存入向量库；
- 新请求进来：先从向量库检索相似历史案例 + 从 Mem0 拉取用户偏好；
- 两者一起作为增强上下文灌入 Prompt；
- LLM 参考「历史案例 + 用户固有喜好」生成高度个性化行程。

---

## 整体落地节奏（带 Mem0 版总结）
1. 不一次性堆全组件，按 V1→V2→V3→V4 渐进迭代；
2. V1 优先跑通：**LangGraph + Redis + Mem0 + MCP**，实现会话记忆、用户长期记忆、工具调用闭环；
3. V2 拆分 Skill、搭建 Workflow 业务编排；
4. V3 加 Evaluation + Loop，实现方案自我打分、循环优化；
5. V4 数据积累后再上向量库，和 Mem0 形成「**结构化用户记忆 + 非结构化案例检索**」双长期记忆架构；
6. 既能快速上线可用，又全程预留后续扩展空间。
