# 旅游AI系统 · 标准PRD文档（业务说明 + 交互细节）
**适用场景**：前端开发 / 后端开发 / UI / 测试 直接使用
**版本**：v1.0
**风格**：简洁、可落地、无歧义

---

## 1 用户与权限模块
### 1.1 微信小程序登录
**业务说明**
用户进入小程序自动完成微信授权登录，后端通过 code 换取 openId，未注册用户自动注册，返回 JWT Token 用于接口鉴权。

**交互细节**
- 打开小程序自动触发 wx.login 获取 code
- 无需用户输入账号密码，全程静默授权
- 登录成功缓存 token 到本地，直接进入首页
- 登录失败弹窗提示：授权失败，请重试
- 除登录接口外，所有请求必须携带请求头：`Authorization: Bearer {token}`
- token 过期/无效时，接口返回 401，前端自动跳转到登录重授权

### 1.2 获取/更新用户信息
**业务说明**
展示用户昵称、头像、旅行偏好设置，支持修改个人信息与偏好（预算、行程节奏）。

**交互细节**
- 进入个人中心自动拉取用户信息并渲染
- 预算选项：低 / 中 / 高；行程节奏：休闲 / 正常 / 紧凑
- 修改信息点击保存，实时调用更新接口
- 保存成功轻提示：更新成功
- 不强制完善信息，允许部分字段为空

### 1.3 管理员：角色/菜单/用户管理
**业务说明**
管理员可查看系统角色、权限菜单树，查询用户列表，支持禁用/启用用户账号。

**交互细节**
- 仅 `ROLE_ADMIN` 角色可访问
- 用户列表强制分页，支持按昵称、openId 模糊搜索
- 禁用后用户无法登录，启用后恢复正常
- 状态切换需二次确认弹窗
- 菜单树根据权限自动渲染，无权限菜单不显示

## 2 行程任务管理（核心业务）
### 2.1 创建行程规划任务
**业务说明**
用户输入旅游需求，系统创建异步任务，由 Python 服务生成结构化行程单。

**交互细节**
- 支持文字输入 + 语音 base64 上传
- 提交按钮做防重复点击（3 秒内不可重复提交）
- 提交成功跳转进度页，展示任务状态
- 无网络时弹窗：网络异常，请检查后重试
- 任务进入队列后，关闭页面不影响后台执行

### 2.2 查询任务状态 & 实时进度推送
**业务说明**
通过 taskId 查询任务状态，WebSocket 实时推送进度、结果、异常。

**交互细节**
- 进度页自动连接 WebSocket
- 状态流转：INIT → QUEUED → PROCESSING → COMPLETED / FAILED
- 实时展示步骤文案：检索攻略 → 规划路线 → 生成行程
- 处理超时自动标记为 FAILED
- 失败页面提供**重试**按钮，可重新发起任务

### 2.3 管理员：重试/取消任务
**业务说明**
对失败任务进行重试，对执行中任务手动取消，终止异步流程。

**交互细节**
- 仅管理员可操作
- 重试后任务状态重置为 QUEUED
- 取消后状态改为 CANCELLED，不可恢复
- 操作成功后自动刷新列表

### 2.4 行程单查看/删除/历史列表
**业务说明**
展示完整结构化行程，支持删除行程、查看历史记录。

**交互细节**
- 行程按天展示，支持左右滑动切换日期
- 历史行程列表强制分页，支持标题搜索
- 删除行程需二次确认弹窗
- 空列表展示空状态页面
- 点击行程卡片进入详情页

## 3 模型会话（AI 对话）
### 3.1 发送对话消息
**业务说明**
用户基于当前行程上下文追问/修改需求，AI 返回回答，可同步更新行程。

**交互细节**
- 可携带 sessionId 保持上下文
- 可绑定 taskId / planId 关联当前行程
- 发送后显示“AI 思考中”状态
- 回复完成自动滚动到底部
- 若修改行程，自动返回新行程结构并刷新页面

### 3.2 对话历史/会话列表
**业务说明**
查看单会话聊天记录，查看用户所有对话会话。

**交互细节**
- 会话列表按最后消息时间倒序排列
- 历史消息默认分页 20 条
- 进入会话自动加载最新 20 条
- 下拉加载更早历史消息

## 4 知识库管理（管理员）
### 4.1 文档上传与向量化
**业务说明**
上传 PDF/Markdown/TXT 文档，后端自动解析、分片、入库、向量化，用于 RAG 检索。

**交互细节**
- 支持点击选择 / 拖拽上传
- 上传显示进度条
- 上传完成展示 docId、切块数量
- 仅管理员可上传
- 重名文件不覆盖，自动加后缀区分

### 4.2 文档列表/删除/向量检索测试
**业务说明**
查看已上传文档，删除文档同步删除向量，支持关键词检索测试。

**交互细节**
- 列表分页展示，支持标签筛选
- 删除需二次确认
- 检索测试输入关键词，返回相似度与内容片段
- 低于相似度阈值的结果不展示

## 5 监控、日志、系统配置
### 5.1 全链路日志查询
**业务说明**
按 traceId / taskId / userId / 时间 / 日志级别查询系统日志，用于问题排查。

**交互细节**
- traceId 与 taskId 至少填写一项
- 日志按时间倒序展示
- ERROR 级别日志标红突出显示
- 支持一键复制 traceId

### 5.2 业务统计与看板
**业务说明**
展示 Token 消耗、任务量、热门城市、关键词统计，用于运营监控。

**交互细节**
- 支持按天 / 按小时聚合数据
- 默认展示近 7 天数据
- 顶部数据卡片自动刷新
- 数字、图表清晰可读

### 5.3 系统健康检查
**业务说明**
实时监控 MySQL、Redis、RabbitMQ、Python Agent 状态与响应延迟。

**交互细节**
- 状态：UP=绿色，DEGRADED=黄色，DOWN=红色
- 每 10 秒自动刷新一次
- 异常时标红并显示异常组件

### 5.4 动态修改 Agent 配置
**业务说明**
在线修改 systemPrompt、temperature、ragTopK、maxTokens，实时生效。

**交互细节**
- 先拉取展示当前配置
- 修改后保存立即生效
- 无需重启服务
- 保存成功轻提示

### 5.5 Redis / 向量库管理
**业务说明**
管理员查看、删除 Redis key，清理向量库数据。

**交互细节**
- 高危删除操作必须二次确认
- 支持批量删除
- 删除成功返回影响条数
- 禁止全量清空操作

## 6 WebSocket 实时推送
**业务说明**
任务进度、完成、失败、系统告警实时推送到前端。

**交互细节**
- 前端使用 STOMP 协议连接 `/ws/websocket`
- 订阅地址：`/user/{userId}/task/{taskId}`
- 推送类型：PROGRESS / RESULT / ERROR
- 网络断开自动重连
- 收到 RESULT 推送自动跳转到行程详情

## 7 全局通用规则（所有页面必须遵守）
**业务说明**
统一系统规范，保证稳定、安全、可追溯。

**交互细节**
- 所有列表接口必须传 `pageNum`、`pageSize`
- 请求头统一携带 `X-Trace-Id`
- 响应格式统一：`code / message / traceId / data`
- 401：自动跳登录
- 403：弹窗提示无权限
- 429：提示请求过于频繁，请稍后再试
- 500：提示服务异常，请稍后重试
