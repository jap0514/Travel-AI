1、使用JWT来生成Token，就不需要存Token了，并且配合Redis做黑名单，当用户退出登录后，如果Token还没有过期，就把该Token放到Redis黑名单中，当再次使用到该Token的时候就先去Redis进行判断。



2、为什么将Token存到Redis时，使用StringRedisTemplate，而不是RedisTemplate？

RedisTemplate会默认JDK序列化，存进去会变成乱码二进制，不适合存Token、黑名单

StringRedisTemplate继承自RedisTemplate，key和value都是String类型，存进去是明文，看得懂，专门用来存Token、验证码、黑名单





3、Redis序列化升级处理：

用 `GenericJackson2JsonRedisSerializer` 自动带入类型信息

- **原因**：使用 `Jackson2JsonRedisSerializer` 存储的对象在取出时会变成 `Map`(反序列化后默认变成 LinkedHashMap)，手动逆串化比较麻烦。

- **升级**：直接改用 `GenericJackson2JsonRedisSerializer`，会在JSON中自动嵌入`@class`，取出时可以直接得到对象。

- ```json
  {    
      "@class": "com.example.entity.User",
   	 "name": "张三",
       "age": 20 
  }
  ```






4、@NotNull`、`@NotBlank

**`@NotNull`**

- **适用类型**：任何类型（String、集合、数组、包装类、自定义对象等）。
- **校验逻辑**：对象不能为 `null`。
- **可通过的情况**：空字符串 `""`、空集合、空白字符串 `" "` 都认为**通过**（因为不是 `null`）。

**`@NotBlank`**

- **适用类型**：**仅适用于字符串**（`CharSequence` 及其子类，如 `String`）。
- **校验逻辑**：`null` 或 **去除首尾空白字符后长度为 0** 都不通过。
- **可通过的情况**：字符串包含至少一个**非空白字符**。例如 `" a "`（去除空白后 `"a"`）通过，`""` 或 `" "` 不通过。
