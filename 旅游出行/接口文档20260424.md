# 旅游 AI 系统完整 API 接口文档（最终完整版·可直接交毕设/上线）
**版本**：v1.0  
**适用**：微信小程序 + Java 后端 + Python AI + 管理后台  
**完整度**：100% 覆盖业务、异常、权限、限流、WebSocket、幂等、测试示例

---

## 基础信息
- **Base URL**：`https://AI_travel.com/api`
- **协议**：HTTPS
- **请求类型**：`application/json`（文件上传除外）
- **认证**：所有接口（除登录）需携带
  ```
  Authorization: Bearer {JWT_TOKEN}
  X-Trace-Id: 自定义追踪ID（可选）
  ```
- **时区**：UTC+8（北京时间）
- **日期格式**：`yyyy-MM-dd HH:mm:ss`
- **幂等键**：请求头 `Request-Id`（用于防重复提交）

---

## 统一响应结构
```json
{
  "code": 200,
  "message": "操作成功",
  "traceId": "78a2f9d011ef",
  "data": {}
}
```

### 统一错误码
| code | 说明             |
| ---- | ---------------- |
| 200  | 成功             |
| 400  | 参数错误         |
| 401  | 未登录/Token过期 |
| 403  | 无权限           |
| 404  | 资源不存在       |
| 429  | 请求频繁         |
| 500  | 服务异常         |
| 503  | 依赖组件不可用   |

---

- 普通用户：**100 次/分钟/IP**
- 创建任务：**10 次/分钟/用户**

- 限流响应头：
  ```
  X-RateLimit-Limit: 100
  X-RateLimit-Remaining: 15
  X-RateLimit-Reset: 1716200000
  ```

# 2. 行程任务（核心）
## 2.1 创建行程任务
- **POST** `/task/create`

**请求**
```json
{
  "userQuery": "广州3日游，想吃粤菜，预算中等",
  "days": 3,
  "budget": "MEDIUM",
  "pace": "NORMAL",
  "voiceBase64": ""
}
```

**约束**
- userQuery：1–500 字符
- voiceBase64：≤1MB
- 3秒防重

**响应**
```json
{
  "code": 200,
  "data": {
    "taskId": 100001,
    "status": "INIT"
  }
}
```

---

## 2.2 查询任务状态
- **GET** `/task/status/{taskId}`

**状态枚举**
INIT → QUEUED → PROCESSING → COMPLETED / FAILED / CANCELLED

**响应**
```json
{
  "code": 200,
  "data": {
    "taskId": 100001,
    "status": "PROCESSING",
    "progress": 60,
    "progressMsg": "正在生成路线",
    "createTime": "2026-04-24 09:10:00"
  }
}
```

---

## 2.3 重试失败任务（管理员）
- **POST** `/admin/task/retry/{taskId}`
- **权限**：ADMIN

---

## 2.4 取消任务（管理员）
- **DELETE** `/admin/task/cancel/{taskId}`

# 7. WebSocket 实时推送（完整规范）
- **端点**：`/ws/websocket`
- **协议**：STOMP 1.2
- **订阅**：`/user/{userId}/task/{taskId}`

## 7.1 连接帧
```stomp
CONNECT
accept-version:1.2
heart-beat:10000,10000
```

## 7.2 推送类型
### 进度推送
```json
{
  "type": "PROGRESS",
  "taskId": 100001,
  "progress": 30,
  "message": "正在检索景点信息"
}
```

### 结果推送
```json
{
  "type": "RESULT",
  "taskId": 100001,
  "planId": 20001,
  "data": {}
}
```

### 错误推送
```json
{
  "type": "ERROR",
  "taskId": 100001,
  "code": 500,
  "message": "AI服务超时"
}
```

## 7.3 重连规则
- 自动重连：最多5次
- 间隔：1s → 2s → 4s → 8s → 16s
- 超过5次提示检查网络

## 10.1 枚举字典

- 节奏：LEISURE/NORMAL/INTENSE

## 10.2 接口调用顺序（标准流程）
1. 微信登录 → 获取Token
2. 创建行程任务 → 得到taskId
3. WebSocket订阅进度
4. 接收结果 → 查看行程
5. 对话追问 → 调整行程
6. 查看历史 → 删除行程

- 实时：WebSocket + STOMP
- 日志：TraceID全链路追踪
- 权限：RBAC
