## 负责人特别叮嘱 (Best Practices)
 1. **关于 traceId 的传递：** 前端（小程序）在调用所有业务接口时，建议在 Header 中带上 X-Trace-ID（如果前端生成了），如果没有，后端生成后应在响应头返回。
 2. **关于数据管理接口的限制：** 向量数据库和 Redis 的数据管理接口（如 DELETE /api/kb/vector/{id}），**必须**限制访问权限，仅限 ADMIN 角色操作，防止误删知识库。
 3. **分页处理：** 对于 GET /api/plan/list 和 GET /api/sys/logs，**必须**强制执行 pageNum 和 pageSize 参数，否则数据量大时会拖垮数据库。















# 旅游 AI 系统完整 API 接口文档 (v1.0)

## 基础信息

- **Base URL**：`https://your-domain.com/api`
- **认证方式**：除登录接口外，所有请求需在 Header 中携带 `Authorization: Bearer <JWT_TOKEN>`
- **全链路追踪**：建议前端在请求头中传递 `X-Trace-Id`（可选），后端会生成并返回在响应头 `X-Trace-Id` 和响应体 `traceId` 字段中
- **响应格式**：统一 JSON 结构（见下方）
- **WebSocket 端点**：`/ws/websocket`（STOMP 协议），连接后需发送 `CONNECT` 帧，然后订阅 `/user/{userId}/task/{taskId}` 或 `/user/{userId}/notice`

### 标准响应结构

```json
{
  "code": 200,
  "message": "success",
  "traceId": "trace_abc123",
  "data": { ... }
}
```

| 字段    | 类型   | 说明                                                         |
| ------- | ------ | ------------------------------------------------------------ |
| code    | int    | 200成功，400客户端错误，401未授权，403无权限，404不存在，500服务器错误 |
| message | string | 提示信息                                                     |
| traceId | string | 全链路追踪ID                                                 |
| data    | object | 业务数据，无数据时返回空对象                                 |

### 错误码说明（code 非 200 时的常见值）

| code | 说明               |
| ---- | ------------------ |
| 401  | Token 无效或过期   |
| 403  | 无操作权限（RBAC） |
| 404  | 资源不存在         |
| 429  | 请求过于频繁       |
| 500  | 内部服务错误       |

- **权限**：`ROLE_ADMIN`

**请求参数**：`pattern`（可选，默认 `*`）

**响应 data**：返回匹配的 key 列表（分页）

- **路径**：`/sys/redis/delete`
- **方法**：`DELETE`
- **权限**：`ROLE_ADMIN`

**请求体**：

```json
{
  "keys": ["task:timeout:task_123", "user:session:456"]
}
```

### 6.5 向量数据库管理（管理员）

- **路径**：`/sys/vector/collections`
- **方法**：`GET`
- **权限**：`ROLE_ADMIN`

**响应 data**：集合列表

- **路径**：`/sys/vector/delete`
- **方法**：`DELETE`
- **权限**：`ROLE_ADMIN`

**请求体**：

```json
{
  "collection": "knowledge_base",
  "ids": ["id1", "id2"]
}
```

## 7. WebSocket 实时通知规范

- **连接端点**：`/ws/websocket`（使用 STOMP 子协议）
- **建立连接后需发送**：

```stomp
CONNECT
accept-version:1.2
heart-beat:10000,10000

^@
```

- **订阅个人任务**：

```stomp
SUBSCRIBE
id:sub-0
destination:/user/{userId}/task/{taskId}
^@
```

- **或订阅全局通知（仅管理员）**：`/topic/admin/alerts`

### 推送消息格式（JSON）

**进度推送**：

```json
{
  "type": "PROGRESS",
  "taskId": "task_123",
  "step": "RAG检索",
  "message": "正在查找相关景点信息",
  "timestamp": "2024-01-01T10:00:00Z"
}
```

**结果推送**（最终完成时）：

```json
{
  "type": "RESULT",
  "taskId": "task_123",
  "planId": 1001,
  "data": { ... }   // 同 /plan/{planId} 结构
}
```

**错误推送**：

```json
{
  "type": "ERROR",
  "taskId": "task_123",
  "code": 500,
  "message": "AI 服务超时"
}
```

## 8. 附录：状态码及状态枚举

### 任务状态枚举

| 状态       | 说明                          |
| ---------- | ----------------------------- |
| INIT       | 刚收到请求，未发送MQ          |
| QUEUED     | 消息已进入 RabbitMQ，等待处理 |
| PROCESSING | Python 正在处理               |
| COMPLETED  | 完成并持久化                  |
| FAILED     | 失败（可重试）                |
| CANCELLED  | 被管理员取消                  |

### HTTP 状态码（实际使用中遵循标准）

| 状态码 | 含义                         |
| ------ | ---------------------------- |
| 200    | 成功                         |
| 400    | 请求参数错误                 |
| 401    | 未认证或Token失效            |
| 403    | 无权限（角色不足）           |
| 404    | 资源不存在                   |
| 429    | 请求频率过高（限流）         |
| 500    | 服务器内部错误，请稍后重试   |
| 503    | 服务不可用（如依赖组件宕机） |
