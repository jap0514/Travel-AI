# 一、用户与权限 —— 业务规则 & 边界条件
## 1. 微信登录
### 业务规则
- 一个微信 openId 只能绑定一个用户
- 首次登录自动注册，不允许重复注册
- Token 有效期固定，过期必须重新登录
- 普通用户只有自己数据的操作权限
- 管理员拥有全部数据操作权限

### 边界条件
- code 无效/已被消费 → 返回 400
- Token 过期 → 返回 401，强制重登
- 无网络 → 不发起登录请求
- 同一用户并发登录 → 最新 Token 有效，旧 Token 自动失效

---

## 2. 用户信息
### 业务规则
- 昵称允许为空，不强制
- 预算默认：medium，节奏默认：normal
- 个人信息只能修改自己的
- 管理员不可修改普通用户昵称/头像

### 边界条件
- 昵称超长 → 自动截取或拦截
- 偏好传入不合法值 → 后端拒绝并返回 400
- 非本人修改他人信息 → 403 无权限

---

# 二、行程任务核心 —— 业务规则 & 边界条件
## 1. 创建行程任务
### 业务规则
- 一次请求只能创建**一个任务**
- 用户必须登录才能创建任务
- 任务进入 MQ 后不允许回滚
- 同一用户**3秒内不可重复提交**
- 语音必须 base64 编码，大小限制 ≤ 1MB

### 边界条件
- userQuery 为空 → 拦截，不允许提交
- 超过频率限制 → 429
- 服务繁忙/队列满 → 返回 QUEUED，不失败
- 语音格式非法 → 忽略语音，只使用文字
- 未传偏好 → 使用用户默认偏好

---

## 2. 任务状态流转（强规则）
### 业务规则
- 状态只能单向流转：
  **INIT → QUEUED → PROCESSING → COMPLETED / FAILED**
- CANCELLED 只能由管理员触发
- FAILED 任务允许重试
- COMPLETED 不可重试、不可取消

### 边界条件
- 查询不存在的 taskId → 404
- 任务已完成仍重试 → 400
- 任务已取消仍查询进度 → 返回 CANCELLED
- 处理超时 → 自动置为 FAILED

---

## 3. 行程单
### 业务规则
- 一个任务对应**0 或 1 个行程单**
- 行程单生成后永久保存，直到用户删除
- 用户只能查看/删除自己的行程单
- 列表接口**必须分页**，禁止全表查询

### 边界条件
- planId 不存在 → 404
- 查看他人行程 → 403
- 行程内容为空 → 不生成行程单
- 删除已删除行程 → 不报错，返回成功

---

# 三、AI 对话（模型会话）—— 业务规则 & 边界条件
## 1. 发送消息
### 业务规则
- 无 sessionId 则自动创建新会话
- 会话与用户绑定，不允许跨用户访问
- 消息长度限制：≤ 500 字符
- 上下文最多保留最近 10 轮

### 边界条件
- 消息为空 → 拦截不发送
- 会话不存在 → 自动新建
- 关联不存在的 taskId → 忽略关联
- 连续快速发送 → 节流拦截

---

## 2. 对话历史
### 业务规则
- 只返回当前用户自己的会话
- 历史消息不允许修改/删除
- 分页默认 20 条，最大 100 条

### 边界条件
- sessionId 非法 → 404
- 翻页超出最大页数 → 返回空列表

---

# 四、知识库管理（管理员）—— 业务规则 & 边界条件
## 1. 文档上传
### 业务规则
- 仅管理员可上传
- 支持：PDF / Markdown / TXT
- 单文件大小 ≤ 10MB
- 上传后自动分片、向量化

### 边界条件
- 文件格式非法 → 400
- 文件过大 → 截断或拒绝
- 重名文件 → 自动重命名，不覆盖
- 向量入库失败 → 标记状态为 FAILED

---

## 2. 向量检索
### 业务规则
- 默认 topK=5，相似度阈值 0.7
- 低于阈值不返回
- 仅管理员可测试

### 边界条件
- query 为空 → 400
- 无匹配内容 → 返回空列表
- 系统异常 → 500

---

# 五、系统监控 & 配置 —— 业务规则 & 边界条件
## 1. 日志查询
### 业务规则
- traceId 和 taskId **至少传一个**
- 只允许管理员查询
- 时间范围最大 7 天

### 边界条件
- 两个参数都不传 → 400
- 时间范围超过 7 天 → 自动截断
- 无日志 → 返回空列表

---

## 2. 系统健康检查
### 业务规则
- 任何组件 DOWN → 整体状态 DEGRADED/DOWN
- 心跳 30 秒未更新 → 标记异常

### 边界条件
- 监控服务不可用 → 返回 503
- 组件延迟过高 → 标黄警告

---

## 3. Agent 配置
### 业务规则
- 修改后**实时生效**，无需重启
- 数值必须在合法范围：
  - temperature：0 ~ 1
  - ragTopK：1 ~ 20
  - maxTokens ≤ 4000

### 边界条件
- 传入非法数值 → 400 拒绝
- 非管理员修改 → 403

---

# 六、Redis / 向量库 —— 业务规则 & 边界条件
### 业务规则
- 高危操作必须**二次确认**
- 不允许执行 flushall / 清空集合
- 只允许按 key / id 批量删除

### 边界条件
- 删除不存在 key → 不报错
- 批量删除最多 50 条
- 越权操作 → 403

---

# 七、WebSocket 实时推送 —— 业务规则 & 边界条件
### 业务规则
- 每个 taskId 只能被对应用户订阅
- 断开连接自动重连最多 5 次
- 完成/失败推送只发一次

### 边界条件
- 无权限订阅 → 直接拒绝
- 任务不存在 → 推送 ERROR
- 超过 5 次重连失败 → 提示检查网络

---

# 八、全局统一业务规则（所有模块必须遵守）
1. **接口必须带 traceId**，前端不传则后端生成
2. **所有列表必须分页**，禁止全表扫描
3. **防重放**：同一请求3秒内只允许一次
4. **越权访问一律 403**
5. **资源不存在一律 404**
6. **参数错误一律 400**
7. **未登录一律 401**
8. **系统异常一律 500**
9. **敏感操作必须二次确认**
10. **不允许返回敏感信息**（如数据库异常栈）
